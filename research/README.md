# Protocol Research Engine (Module 15 + 16)

Автоматизированный discovery новых DeFi протоколов для аирдроп-фарминга. Работает еженедельно, находит и оценивает протоколы с потенциалом аирдропа.

---

## Файлы модуля

### `news_aggregator.py`

**Зачем нужен:**  
Находит кандидатов для фарминга — собирает информацию о новых протоколах из внешних источников.

**Бизнес-задача:**  
Агрегирует данные из 5 источников (Crypto News API, DefiLlama, RSS: The Block, CoinDesk, L2Beat). Выявляет протоколы без токена (признак будущего аирдропа) и фокусируется на L2-сетях.

**Встраивание в систему:**  
Входная точка Research Engine. Данные передаются в `NewsAnalyzer` для фильтрации, затем в `ProtocolAnalyzer` для LLM-оценки.

**Ключевые особенности:**
- Защита от блокировок: rate limiting, user-agent rotation, exponential backoff
- Целевые цепи загружаются из БД (Smart Risk Engine), не захардкожены
- Приоритет протоколам без токена (высокий потенциал аирдропа)

**Заглушки:** Нет  
**Хардкод:** User-agent список, лимиты запросов (5-10 req/min)

---

### `news_analyzer.py`

**Зачем нужен:**  
Снижает затраты на LLM — отсеивает нерелевантные статьи до дорогого анализа.

**Бизнес-задача:**  
Фильтрует новости по ключевым словам (airdrop, points program, L2, token launch). Экономит 70-80% LLM API costs. Присваивает score релевантности (0-100), blacklist отсекает спам и регуляторные новости.

**Встраивание в систему:**  
Промежуточный слой между `NewsAggregator` и `ProtocolAnalyzer`. Вызывается в `scheduler.py` перед LLM-анализом.

**Ключевые особенности:**
- Порог релевантности: ≥60 баллов проходят дальше
- 4 группы ключевых слов с весами (airdrop_direct: 40, points_programs: 35, l2_ecosystem: 15, token_launch: 25)
- Blacklist: SEC, lawsuit, regulation, hack, scam и др.

**Заглушки:** Нет  
**Хардкод:** Ключевые слова и blacklist в коде (не в БД)

---

### `protocol_analyzer.py`

**Зачем нужен:**  
Принимает решение — стоит ли фармит этот протокол.

**Бизнес-задача:**  
LLM (GPT-4 Turbo) анализирует протокол и присваивает:
- Airdrop score (0-100)
- Рекомендуемые действия (SWAP, LP, STAKE, BRIDGE)
- Оценку рисков и reasoning
- Проверяет доступность бриджа для цепи протокола

Протоколы со score ≥70 попадают в очередь на approval.

**Встраивание в систему:**  
Финальная стадия Research Engine. Результаты сохраняются в `protocol_research_pending` для human-in-the-loop подтверждения через Telegram.

**Ключевые особенности:**
- Стоимость: ~$0.01-0.03 за протокол
- Проверка бриджей через Bridge Manager v2.0
- Классификация протоколов: сначала DefiLlama API, fallback на LLM
- При невозможности классификации — алерт в Telegram для ручной обработки

**Заглушки:** Нет  
**Хардкод:** Модель GPT-4 Turbo, цены API, промпты в коде

### `scheduler.py`

**Зачем нужен:**  
Оркестрирует весь процесс discovery — запускает цепочку агрегация → фильтрация → анализ.

**Бизнес-задача:**  
Еженедельный автоматизированный поиск новых протоколов (понедельник 00:00 UTC). Координирует пайплайн, ведёт логи, отправляет результаты в Telegram.

**Встраивание в систему:**  
Вызывается `MasterOrchestrator`. Управляет потоком данных между всеми компонентами Research Engine.

**Пайплайн:**
1. `NewsAggregator` — сбор кандидатов
2. Фильтрация за последние 30 дней
3. `NewsAnalyzer` — отсев нерелевантных
4. `ProtocolAnalyzer` — LLM-оценка
5. Сохранение в `protocol_research_pending`
6. Telegram-уведомление

**Ключевые особенности:**
- Human-in-the-loop: протоколы требуют подтверждения через Telegram
- Трекинг затрат на LLM
- Дедупликация протоколов

**Заглушки:** Нет  
**Хардкод:** Фильтр 30 дней

## Архитектура потока

```
Внешние источники (API, RSS)
         ↓
  NewsAggregator (сбор)
         ↓
  NewsAnalyzer (фильтр 70-80% экономии LLM)
         ↓
  ProtocolAnalyzer (LLM-оценка $0.01-0.03/протокол)
         ↓
  protocol_research_pending (БД)
         ↓
  Telegram → Human approval
```

---

## Интеграция с системой

| Компонент | Взаимодействие |
|-----------|----------------|
| **База данных** | Читает целевые цепи, пишет кандидатов, логи исследований |
| **MasterOrchestrator** | Вызывает `run_research_cycle()` еженедельно |
| **Telegram Bot** | Уведомления о результатах, алерты при ошибках |
| **Bridge Manager** | Проверка доступности бриджей для новых протоколов |
| **Smart Risk Engine** | Предоставляет список активных цепей для фарминга |

---

## Критичные моменты

1. **Хардкод ключевых слов** в `news_analyzer.py` — при изменении терминологии нужно обновлять код
2. **Зависимость от OpenRouter** — при недоступности API весь анализ останавливается
3. **Нет fallback на другие LLM** — только GPT-4 Turbo через OpenRouter
4. **Ключевые слова не в БД** — нельзя обновлять без деплоя
